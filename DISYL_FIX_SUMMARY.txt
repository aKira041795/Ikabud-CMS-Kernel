  DiSyL Template Rendering - Root Cause Fix Summary
═══════════════════════════════════════════════════════════════════════════════

PROBLEM:
  DiSyL templates produced no output despite successful compilation.
  Expressions like {title} and {excerpt} were not being interpolated.

ROOT CAUSE:
  Parser treated {title} as a component tag instead of a variable expression.
  No "expression" node type existed in the AST.
  Renderer had no logic to evaluate and output variable values.

SOLUTION:
  1. Enhanced Parser to detect expressions vs component tags
  2. Added 'expression' node type to AST
  3. Implemented renderExpression() in BaseRenderer and WordPressRenderer
  4. Expression detection rules:
     - No attributes
     - No ikb_ prefix
     - Lowercase first char OR contains dots (item.title)

FILES MODIFIED:
  ✓ /kernel/DiSyL/Parser.php
  ✓ /kernel/DiSyL/Renderers/BaseRenderer.php
  ✓ /kernel/DiSyL/Renderers/WordPressRenderer.php

EXPRESSION SYNTAX:
  {title}           → Simple variable from context
  {item.title}      → Nested property (dot notation)
  {item.excerpt}    → Object/array property access
  {categories}      → Arrays rendered as comma-separated

EXAMPLE USAGE:
  {ikb_query type="post" limit="3"}
      {ikb_card}
          {ikb_text size="lg"}{item.title}{/ikb_text}
          {ikb_text size="sm"}{item.excerpt}{/ikb_text}
      {/ikb_card}
  {/ikb_query}
